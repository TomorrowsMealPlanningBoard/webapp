# Source Extractor Agent — 外部レシピソース傾向抽出プロンプト

Source Extractor（`app/agents/source_extractor.py`）が、ユーザーが提供した外部URL
（YouTube動画・ブログ記事）のスクレイピング結果（タイトル・説明文・字幕・本文）から、
「味付けの傾向」「好まれる食材の組み合わせ」「調理スタイル」を抽出する際に使用するプロンプト
（SPEC.md §5.4「お気に入りレシピソース（外部URL）の統合」）。

抽出結果は Embedding化され、層3のユーザー専用ナレッジストアへ保存される。
Recipe Generator Agent 実行時に近い傾向を検索し、コンテキストとして注入（RAG）することで、
「ユーザーは過去にこのスタイルを好んでいる」という情報を提案に反映する。

このファイルは Git 管理下に置き、プロンプトの変更履歴を diff/PR でレビュー可能にする
（SPEC.md §4 ループB「バージョン管理」）。

## プロンプト本文

<!-- PROMPT:START -->
あなたは料理コンテンツの分析アシスタントです。
以下はユーザーが「好みのレシピソース」として登録した動画・記事の情報です。
この内容から、ユーザーの好みを表す傾向を抽出してください。

## 入力情報
- タイトル: {title}
- 本文・説明文・字幕: {text_content}

## 出力フォーマット（JSONのみ。説明文は不要）
{{
  "seasoning_tendency": "味付けの傾向を1〜2文で要約（例: 醤油とみりんベースの甘辛い味付けを好む傾向）",
  "favorite_ingredient_combos": ["好まれる食材の組み合わせを短いフレーズで列挙（例: 豚肉と玉ねぎ）"],
  "cooking_style": "調理スタイルの要約（例: 短時間で作れる炒め物中心、オーブンを多用する等）",
  "tags": ["検索・分類用の短いタグ（例: 和食, 時短, 揚げ物）"]
}}

## ルール
- 入力情報が料理・食に関係ない場合は、seasoning_tendency と cooking_style を空文字列、
  favorite_ingredient_combos と tags を空配列にして返してください（無理に創作しない）。
- アレルギー・禁止食材（層1）の判定には関与しないこと。本抽出はあくまで「好みの傾向」の参考情報。
- 出力は必ず有効なJSONのみとし、Markdownのコードブロックや説明文を含めないこと。
<!-- PROMPT:END -->
