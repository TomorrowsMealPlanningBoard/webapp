# TomorrowsMeal 開発ガイド & デザインルール

## 0. 実装着手前に必ず行うこと

1. **対象チケットのACを確認する**
   - ACが記載されていない場合は、**実装を開始せず**、まずACの案を作成してユーザーに提示し、承認を得てからチケットに記載する。
2. **チケットのステータスをIn Progressに変更する**
3. **設計判断に迷ったら [SPEC.md](SPEC.md) を参照する**（アーキテクチャ・データ設計・エージェント構成の根拠）
4. **層1（アレルギー・禁止食材）は決定的フィルタで実装する**（ベクトル検索や確率的処理を使わない。if文による機械的除外のみ）

開発コマンド・テスト実行方法・実装完了の定義は [CONTRIBUTING.md](CONTRIBUTING.md) を参照。

## 1. テックスタック・前提条件
- **CSS フレームワーク**: Tailwind CSS
- **UI コンポーネントライブラリ**: daisyUI
- **レスポンシブ対応**: 完全モバイルファースト（幅375px〜420px程度のスマートフォン画面が基準。PCでは `max-w-md w-full` でセンタリング）

---

## 2. デザインコンセプト & カラーテーマ
- **アプリの世界観**: 「優しくて親しみやすい」「清潔感がある」「冷蔵庫・料理のワクワク感」
- **daisyUI テーマ**: `emerald`（または `cupcake`）をベースとする。
  - プライマリー（緑・エメラルド系）: 新鮮な野菜、料理、健康をイメージ
  - セカンダリー / アクセント: 暖色系、料理の美味しさを引き立てるポップな色
  - 背景色: 清潔感のある明るい白・オフホワイトベース（`bg-base-200` / `bg-base-100`）

---

## 3. 共通UIコンポーネントルール
*原則としてdaisyUIの標準クラスを最大限に活かし、独自のカスタムCSSは書かないこと。*

- **ボタン**: `.btn-primary` / `.btn-secondary`。最小高さ48px相当（`h-12` や `py-3`）を確保。
- **カード**: `.card .card-bordered .bg-base-100 .shadow-xl`（または `shadow-sm`）、`.rounded-box` または `.rounded-2xl`。
- **タイポグラフィ**: 日本語が読みやすいフォント（`Plus Jakarta Sans` や `Noto Sans JP` など、メイリオ等のOS依存フォントは避ける）。料理名や重要ヘッダーは `.text-xl .font-bold .text-base-content`。
- **余白**: コンテナは `.p-4` / `.p-6`、縦方向の並びは `.space-y-4`（または `.space-y-6`）。

---

## 4. 演出・UXルール
- **ローディング**: `.loading .loading-dots .loading-lg` + 「AIが献立を考えています...」のような遊び心のある文言。
- **状態の表現**: 賞味期限切れ・エラー・警告は `.text-error` / `.badge-error` で視覚的に一瞬で伝える。
